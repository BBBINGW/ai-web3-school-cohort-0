## 📖 What I Read: LLM Chapter

**URL:** https://aiweb3.school/zh/handbook/ai/llm/

### Core Thesis

LLM is not a black box that can talk — it's a **probability model** that compresses massive text, code, and multimodal signals into parameters. Know when to trust it and when to verify.

### First Principles

| Principle | Meaning |
|-----------|---------|
| Model output is a candidate, not fact | Everything needs verification before execution |
| Model is a reasoning layer, not a truth source | Key facts must come from DB, API, logs, or trusted systems |
| Make outputs checkable | Summaries, plans, operations → structured schemas, citations |
| Expose uncertainty early | Downgrade gracefully, don't fabricate |

### Key Concepts

| Concept | Core Idea |
|---------|-----------|
| **Token** | Basic processing unit. Affects context window size, cost, and whether the model sees key info |
| **Embedding** | Vector representation for semantic similarity. Good for search/RAG, not for truth verification |
| **Transformer** | Attention architecture — great at pattern matching, not a stable database |
| **Hallucination** | Plausible but wrong outputs → don't fight with better prompts, fight with external verification |
| **Multimodal** | Text + image + audio input. Helps models read screens/diagrams, but still needs boundaries |

### Position in AI×Web3

LLM sits in the **understanding & generation layer**. Real products need:

```
Data layer (RPC, indexers, oracles, vector DB)
→ Orchestration (Prompt, Context, RAG, Agent workflow)
→ Execution (tool calls, wallet, Smart Account, contract interaction)
→ Safety (Guard, simulation, permission policy, audit logs)
```

The closer LLM gets to execution, the more its natural language output must become verifiable objects.

### Connection to Cryptography

Many concepts here directly parallel cryptographic thinking:

| LLM Concept | Crypto Parallel |
|-------------|-----------------|
| Token context window | Block size / transaction calldata |
| Hallucination risk | Byzantine fault — don't trust, verify |
| Embedding similarity ≠ truth | Probabilistic verification ≠ proof |
| Model as reasoning layer | Prover vs. verifier separation |
| Audit trail for LLM outputs | Merkle proof of execution |
