# Modulus Labs 深度拆解

> 拆解时间：2026-05-24
> 项目官网：https://moduluslabs.xyz

---

## 1. 它在解决什么问题

**核心问题：如何让智能合约可信地使用 AI 推理结果？**

链上直接跑 ML 模型太贵了（Gas 费、计算能力限制），所以推理必须在链下做。但链下推理结果上链时，**没法保证可信** — 节点可能返回错误或被篡改的结果。

Modulus 的解法：**用 ZK-SNARK 为 AI 推理生成证明，链上验证证明即可获得可信任结果。** 这同时实现了：
- 去信任化（不需要信任推理方，只需信任证明系统和区块链）
- 隐私保护（证明不泄露模型权重或输入数据）
- 即时验证（链上验证远快于重新跑推理）

---

## 2. AI 部分是什么

将 ML 推理（神经网络前向传播）拆解为 ZK 友好的算术电路。

**支持的模型：**
- 小型神经网络（MLP、CNN）
- 决策树 / 随机森林
- 正在向 Transformer 扩展

**技术路径：**
- 使用 Groth16 / PLONK 等 SNARK 后端
- 将 ML 算子（ReLU、Sigmoid、线性层）编译为 R1CS 约束
- 证明生成在链下 GPU 节点完成（耗时 30s ~ 5min）
- 验证合约是轻量级 Solidity 代码

**当前限制：**
- 大型模型（>100M 参数）的证明生成仍很慢（数十分钟到小时级）
- 尚不支持完整 LLM 推理（GPT-4 规模）
- 暂不支持模型权重隐私保护

---

## 3. Web3 / 区块链部分是什么

**核心合约：** `Verifier.sol`
- 部署在以太坊主网及 L2（Optimism、Arbitrum）
- 输入：AI 推理公开输入 + ZK 证明
- 输出：true/false — 证明有效则接受推理结果

**典型流程：**
```
DApp 请求 → 链下推理节点执行 ML 模型 + 同时生成 ZK 证明
         → 推理结果 + 证明提交至链上 Verifier 合约
         → Verifier 验证证明 → 通过则结果生效
```

**链上消耗示例（来自官方博客）：**
- 小型 MLP（3层，128 神经元）的证明验证 Gas：~200K–400K（< $10 普通 Gas 下）
- 证明生成时间（本地 GPU）：~30 秒–5 分钟

## 5. 与其他方案的对比

| 维度 | ZK (Modulus) | OPML (Optimistic ML) | TEE (机密计算) |
|------|-------------|----------------------|---------------|
| 信任假设 | 纯密码学（无信任） | 经济博弈（挑战期） | 硬件信任（Intel/AMD） |
| 延迟 | 实时验证（~秒级） | 挑战期（小时~天） | 低延迟 |
| 隐私保护 | 强 | 弱（需公开推理结果） | 强 |
| 链上成本 | 中（200K–1M Gas） | 低 | 最低 |
| 链下成本 | 高（GPU 生成证明） | 低 | 中（TEE 配置） |
| 成熟度 | 早期（<10M 参数可行） | 概念验证 | 已有商用 |

## 6. 我的观察与判断

### 6.1 与本周学习的关联

**交叉点直击：** 这周我在做 AI × Web3 交叉流程图时，把"人在环确认"放在 Tool Use 之后、链上交易之前。Modulus 本质上是在解决**如果 Agent 想自动验证链上数据，怎么确保验证结果可信** — 这正是我 README 总结里写的"还没解决的问题"。

**HITL 的替代方案：** 这周我强调"签名必须人工确认"，这是当前最安全的方式。但 Modulus 的思路提示了另一种路径：如果 AI 的推理结果可以通过 ZK 证明自动验证，那**部分低风险操作理论上可以绕过人工确认**，由密码学保证正确性。

### 6.2 可行性判断

对于小型任务型模型（信用评分、交易策略判断），Modulus 的方案已经足够实用。但如果想扩展到 LLM 级别，需要突破性的证明系统优化（GKR、Plonky3 等）。Modulus 团队目前没展示这条路径。

### 6.3 竞争格局

同赛道有 **EZKL**（开源，支持 ONNX 模型）、**Giza**（聚焦 DeFi 模型）。Modulus 目前差异化不大，早期社区关注度是优势，但缺乏核心技术论文作为护城河。

### 6.4 从密码学研究者视角的疑问

一个还没想清楚的问题：ZK 证明系统的**形式化验证**本身也是一个难题。证明系统有 bug 怎么办？「谁验证验证者」这个问题在 Modulus 的架构里似乎没有回答 — 他们假设证明系统本身是正确的，但这跟「信任 Intel SGX 不会泄漏私钥」本质上是一样的信任假设，只是从信任硬件变成了信任数学证明代码的实现。
